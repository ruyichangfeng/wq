微擎0.8配置步骤

1.设置好模块参数设置内的所有参数和选项。

2.在微信公众号后台——公众号设置——功能选项中设置‘JS接口安全域名’，域名为:{php echo trim(str_replace(array('http://','https://'),'',$_W['siteroot']),'/')}

3.在微信公众号后台——公众号设置——功能选项中设置‘网页授权域名’，域名为:{php echo trim(str_replace(array('http://','https://'),'',$_W['siteroot']),'/')}

4.在微擎系统后台——功能选项——支付参数中设置‘微信支付’内的参数。

5.在微信公众号后台——微信支付——开发配置中设置‘支付授权目录’，目录为:{php echo $_W['siteroot'].'payment/wechat/'}

6.在微信公众号后台——基本配置中设置‘IP白名单’，ip是你的服务器ip

团购商城模块使用注意点：

1.设置好模块参数，在参数设置内，根据提示设置，不要设置错误的参数，否则可能导致站点出错。

2.会员账户余额数据使用的是微擎自带的会员余额，微擎会员的余额在本模块内会生效。

3.应对整个模块进行彻底的了解后再正式运营。

4.模块使用的提现支付参数设置是在微擎系统的功能选项-支付参数中的微信支付项设置，模块只配置了微信支付和余额支付。

常见问题

1.为什么无法支付？为什么不能选择地址？

答：请在微信公众号后台设置授权域名，然后在微擎系统的功能选项-支付参数中的微信支付项设置。需要在微信公众号后台设置js安全域名，如果不会请百度如何设置js安全域名。

2.订单和团状态分几种？

答：订单状态有：待支付、已取消、已支付、已发货、已完成、已退款、买家申请退款 。团状态分为：组团中、组团失败、组团成功、团退款中。
当团在 组团中、退款中的时候，团对应的订单不能发货，不能退款。当团失败了、团成功了，对应的订单状态可以发货、退款处理。
买家可操作订单的方式有：取消订单（取消未支付的订单）、申请退款（已支付状态的订单发货后不能申请退款），确认收货（后台发货后），评价商品（订单完成后）。

3.为什么首页没有内容？

答：请在装修页面内设计一个页面，然后回到页面列表里将此页面设为首页。

4.为什么后台打开后很乱？

答：情使用谷歌浏览器或搜狗浏览器，切换到急速模式

5.为什么我提交了退款确没有退款给买家？

答：请设置好计划任务，计划任务是间隔一定时间执行的，可能还没到计划任务执行时间，再等1分钟看看情况。也可能微信支付账户的钱已经被结算到对公账户，没有钱可退款。

6.积分抵扣优惠是怎么计算的？

答：积分抵扣比例在参数设置内设置，具体的商品能抵扣多少在编辑商品的时候设置。当买家选择多款商品或多件同款商品一起支付的时候，积分抵扣只计算一次，并且取最大的抵扣值。比如：买家购买 A 和 B 两种商品，一起支付，A可抵扣10元，B可抵扣5元，那么最终抵扣只取10元。比如：买家购买A款商品5件，A商品设置的抵扣金额是10元，那么即使购买5件也只抵扣10元。

7.首单优惠是怎么计算的？

答：会员第一次在平台购物的时候给予优惠，同款多件和多款商品只计算一次，且取最大值。比如：买家购买 A 和 B 两种商品，一起支付，A首单优惠10元，B首单优惠5元，那么最终首单优惠只取10元。比如：买家购买A款商品5件，A商品设置的首单优惠是10元，那么即使购买5件也只优惠10元。

8.优惠券优惠是怎么计算的？

答：优惠券只对设置了可使用优惠券的商品有效（在编辑商品的时候设置当前商品是否可使用优惠券）。比如：买家购买2件 A(单价10元) 和3件 B (单价10元)商品，一起支付，A可使用优惠券，B不可使用优惠券，买家使用一张9折折扣券，那么最终订单金额是10*2*0.9+10*3=48元。比如：买家购买A款商品5件，A商品价格10元且可使用优惠券，买家使用一张抵扣10元的代金券，那么最终订单金额是5*10-10=40元，也就是说，折扣券对每个可使用优惠券的商品都计算，代金券只计一次。

9.满额包邮是怎么计算的？

答：每款商品的邮费是单独计算的，若用户订单总金额超过了相应商品设置的满额包邮，那这款商品就包邮处理。比如：买家购买 A(单价10元，邮费5元) 和 B (单价10元，邮费10元) 两种商品，假设购买A 3件，B 3件，A满50包邮，B满500包邮，系统会先计算总金额：10*3+10*3=60元，总金额60元达到了A商品满50包邮条件，那么A商品会包邮处理，不满足B商品包邮条件，B商品不会包邮。

10.重要提示！！！！！！！！！！

答：所有优惠都是先计算总金额，在总金额的基础上计算叠加优惠。比如:买家购买A商品(单价10元，购买10件，邮费10元)，假如买家可积分抵扣5元，优惠券9折，首单优惠3元，满200包邮。那么系统先计算订单金额:单价10*数量10 = 订单基础金额100元，然后在100元(不算邮费)的基础上计算优惠:最终的订单金额 = 基础总额100 - 积分抵扣5元 - 优惠券优惠(100*0.1=10元) - 首单优惠3元 = 100-5-10-3+10 = 92元！！！(最后的那个10是邮费，因为是满200包邮，基础总额才100元，不满足包邮条件)

11.为什么后台没有操作管理会员的页面？

答：模块继承微擎自带会员系统，在微擎的粉丝营销-会员内可对相应的会员增减积分和余额。
